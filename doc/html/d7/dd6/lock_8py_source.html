4 '''@package _cookbook._thread
5 Multi-threading using Lock.
6 
7 A primitive lock is a synchronization primitive that is not owned by a
8 particular thread when locked. In Python, it is currently the lowest level
9 synchronization primitive available, implemented directly by the
10 <code>thread</code> extension module.
11 
12 A primitive lock is in one of two states, "locked" or "unlocked". It is
13 created in the unlocked state. It has two basic methods, `acquire()` and
14 `release()`. When the state is unlocked, `acquire()` changes the state to
15 locked and returns immediately. When the state is locked, `acquire()` blocks
16 until a call to `release()` in another thread changes it to unlocked, then the
17 `acquire()` call resets it to locked and returns. The `release()` method
18 should only be called in the locked state; it changes the state to unlocked
19 and returns immediately. If an attempt is made to release an unlocked lock, a
20 `ThreadError` will be raised.
